This tutorial will guide you through processing your first literary text file with CADS. You'll learn how to clean a text file by removing headers, footers, and page numbers.

## Overview

CADS automatically removes common artifacts from literary text files:
- Headers and footers (e.g., book titles, author names)
- Page numbers at the beginning, middle, or end of lines
- Roman numerals at the start of files
- Excessive whitespace

<Steps>
  <Step title="Prepare a Sample Text File">
    Create a sample text file to process. Let's use an example with typical artifacts:

    ```bash
    echo "Cien años de soledad
    
I

Gabriel García Márquez

The world was so recent that many things lacked names,
and in order to indicate them it was necessary to point.

1

Every year during the month of March a family of ragged gypsies
would set up their tents near the village.

2

Gabriel García Márquez

They came with a great uproar of pipes and kettledrums." > sample.txt
    ```

    This sample file contains:
    - A header ("Cien años de soledad") and author name
    - A Roman numeral (I)
    - Page numbers (1, 2)
    - The actual text content
  </Step>

  <Step title="Run CADS">
    Process the file with CADS:

    ```bash
    deno run -A clean_text.ts sample.txt
    ```

    You should see output confirming the operation:
    ```
    Cleaned text has been written to cleaned/sample.txt
    ```

    <Note>
      The `-A` flag grants CADS all necessary permissions to read your input file and write the cleaned output.
    </Note>
  </Step>

  <Step title="View the Results">
    Check the cleaned output:

    ```bash
    cat cleaned/sample.txt
    ```

    The cleaned text will have:
    - Headers and footers removed
    - Page numbers stripped out
    - Roman numerals eliminated
    - Clean, readable text
  </Step>
</Steps>

## Understanding the Output

CADS creates a `cleaned/` directory in your current working directory and saves the processed file there with the same filename as the original.

**Directory Structure:**
```
your-project/
├── clean_text.ts
├── sample.txt          # Your original file
└── cleaned/
    └── sample.txt      # The cleaned version
```

<Note>
  The original file is never modified. CADS always creates a new file in the `cleaned/` directory.
</Note>

## Processing Your Own Files

Now that you've processed a sample file, you can clean your own literary text files:

```bash
deno run -A clean_text.ts path/to/your/book.txt
```

### Best Practices

<Card title="Batch Processing" icon="layer-group">
  Process multiple files by running CADS in a loop:
  
  ```bash
  for file in *.txt; do
    deno run -A clean_text.ts "$file"
  done
  ```
</Card>

### What Gets Removed

CADS is pre-configured to remove specific headers and footers. Looking at the source code (`clean_text.ts:9-12`), you can see the default patterns:

```typescript
const headersAndFooters = [
  "Cien años de soledad",
  "Gabriel García Márquez"
];
```

<Note type="warning">
  **Customization Required**: To clean your own books, you'll need to modify these patterns in `clean_text.ts` to match your specific text files' headers and footers.
</Note>

### Page Number Removal

CADS intelligently removes page numbers in three scenarios:
- After complete sentences (preserves paragraph structure)
- In the middle of paragraphs (maintains text flow)
- At the end of files

The tool uses pattern matching to identify and remove numeric page numbers while preserving the text's readability.

## Common Use Cases

### Literary PDFs Converted to Text

If you've converted a PDF book to text format, it often contains:
- Repeated headers and footers on every page
- Page numbers interrupting text flow
- Chapter numbers in Roman numerals

CADS cleans these artifacts automatically.

### Example Workflow

```bash
# Convert PDF to text (using external tool)
pdftotext book.pdf book.txt

# Clean with CADS
deno run -A clean_text.ts book.txt

# View cleaned output
less cleaned/book.txt
```
