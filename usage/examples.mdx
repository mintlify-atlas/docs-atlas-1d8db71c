## Overview

These examples show how CADS processes actual text from "Cien años de soledad" (One Hundred Years of Solitude) by Gabriel García Márquez. Each example demonstrates a specific cleaning operation.

## Example 1: Removing Headers and Footers

### Input Text (01.md)

Notice the book title and author name appearing in the middle of the text:

```text
suelo. Construyó otro, reunió en el cuartito a los hombres del pueblo y les 
demostró, con teorías que para todos resultaban incomprensibles, la posibilidad 
de regresar al punto de partida navegando siempre hacia el Oriente.



Cien años de soledad



Gabriel García Márquez

Al principio, José Arcadio Buendía era una especie de patriarca juvenil, que 
daba instrucciones para la siembra y consejos para la crianza de niños...
```

### Output Text (cleaned/01.md)

Headers and footers removed, text flows naturally:

```text
suelo. Construyó otro, reunió en el cuartito a los hombres del pueblo y les 
demostró, con teorías que para todos resultaban incomprensibles, la posibilidad 
de regresar al punto de partida navegando siempre hacia el Oriente.
Al principio, José Arcadio Buendía era una especie de patriarca juvenil, que 
daba instrucciones para la siembra y consejos para la crianza de niños...
```

<Accordion title="How It Works">
The header/footer removal uses three passes:

1. **Middle of text**: Replaces header/footer surrounded by text with a newline
2. **After punctuation**: Removes header/footer following sentence endings
3. **End of file**: Completely removes trailing headers/footers

See [Text Processing → Header and Footer Removal](/usage/text-processing#2-header-and-footer-removal) for details.
</Accordion>

## Example 2: Removing Roman Numerals

### Input Text (05.md)

Chapter files often start with Roman numerals:

```text
V 



Aureliano Buendía y Remedios Moscote se casaron un domingo de marzo ante el 
altar que el padre Nicanor Reyna hizo construir en la sala de visitas.
```

### Output Text (cleaned/05.md)

Chapter marker removed, text starts cleanly:

```text
Aureliano Buendía y Remedios Moscote se casaron un domingo de marzo ante el 
altar que el padre Nicanor Reyna hizo construir en la sala de visitas.
```

<Accordion title="Pattern Details">
**Regex**: `^\s*[IVXLCDM]+\s*\n`

- `^` - Start of file
- `\s*` - Optional whitespace
- `[IVXLCDM]+` - One or more Roman numeral characters
- `\s*\n` - Optional whitespace and newline

This removes chapter markers like: I, II, III, IV, V, X, XX, etc.
</Accordion>

## Example 3: Page Numbers After Sentences

### Input Text

Page numbers often appear after complete sentences:

```text
La idea de un Macondo peninsular prevaleció durante mucho tiempo, inspirada 
en el mapa arbitrario que dibujó José Arcadio Buendía al regreso de su expedición.
57
Lo trazó con rabia, exagerando de mala fe las dificultades de comunicación, 
como para castigarse a sí mismo por la absoluta falta de sentido con que eligió 
el lugar.
```

### Output Text

Page number replaced with paragraph break:

```text
La idea de un Macondo peninsular prevaleció durante mucho tiempo, inspirada 
en el mapa arbitrario que dibujó José Arcadio Buendía al regreso de su expedición.

Lo trazó con rabia, exagerando de mala fe las dificultades de comunicación, 
como para castigarse a sí mismo por la absoluta falta de sentido con que eligió 
el lugar.
```

<Note>
The page number after a sentence is replaced with **two newlines** to preserve the paragraph break. This maintains the document's natural structure.
</Note>

## Example 4: Page Numbers in Paragraphs

### Input Text

Sometimes page numbers split paragraphs mid-sentence:

```text
usaba un sombrero grande y negro, como las alas extendidas de un cuervo, y 
un chaleco de terciopelo patinado por el verdín de los siglos. Pero a pesar de 
su inmensa sabiduría y de su ámbito misterioso, tenía un peso humano, una 
condición terrestre que lo



Cien años de soledad



Gabriel García Márquez

mantenía enredado en los minúsculos problemas de la vida cotidiana.
```

### Output Text

Headers removed, paragraph rejoined:

```text
usaba un sombrero grande y negro, como las alas extendidas de un cuervo, y 
un chaleco de terciopelo patinado por el verdín de los siglos. Pero a pesar de 
su inmensa sabiduría y de su ámbito misterioso, tenía un peso humano, una 
condición terrestre que lo
mantenía enredado en los minúsculos problemas de la vida cotidiana.
```

<Accordion title="Why Single Newline?">
When a page break occurs mid-paragraph, CADS uses a **single newline** instead of a paragraph break. This preserves the continuous flow of the text while removing the artifact.

The algorithm distinguishes between:
- **Paragraph breaks** (after period) → Two newlines
- **Mid-paragraph breaks** → Single newline
</Accordion>

## Batch Processing Example

### Process All Chapter Files

```bash
# Process files 01.md through 20.md
for i in {01..20}; do
  echo "Processing chapter $i..."
  deno run -A clean_text.ts "$i.md"
done
```

### Output

```text
Processing chapter 01...
Cleaned text has been written to cleaned/01.md
Processing chapter 02...
Cleaned text has been written to cleaned/02.md
Processing chapter 03...
Cleaned text has been written to cleaned/03.md
...
Processing chapter 20...
Cleaned text has been written to cleaned/20.md
```

<Accordion title="Parallel Processing">
For faster processing of many files, run commands in parallel:

```bash
for i in {01..20}; do
  deno run -A clean_text.ts "$i.md" &
done
wait
echo "All chapters processed!"
```

This runs all 20 commands simultaneously in the background.
</Accordion>

## Advanced Usage Patterns

### Clean and Concatenate

Create a single file from all cleaned chapters:

```bash
# First clean all files
for file in *.md; do
  deno run -A clean_text.ts "$file"
done

# Then concatenate cleaned versions
cat cleaned/*.md > complete_book.md
```

### Clean with Verification

Compare before and after:

```bash
#!/bin/bash

for file in *.md; do
  # Count lines before
  before=$(wc -l < "$file")
  
  # Clean the file
  deno run -A clean_text.ts "$file"
  
  # Count lines after
  after=$(wc -l < "cleaned/$file")
  
  # Calculate reduction
  reduction=$((before - after))
  echo "$file: $before → $after lines (-$reduction)"
done
```

### Output

```text
01.md: 483 → 419 lines (-64)
02.md: 385 → 342 lines (-43)
05.md: 674 → 549 lines (-125)
...
```

## Real-World Workflow

<Steps>
  <Step title="Extract PDF to Text">
    Use a PDF extraction tool to create `.md` files:
    
    ```bash
    pdf-to-text book.pdf --output chapters/
    ```
  </Step>
  
  <Step title="Run CADS on All Files">
    Process the extracted files:
    
    ```bash
    cd chapters/
    for file in *.md; do
      deno run -A clean_text.ts "$file"
    done
    ```
  </Step>
  
  <Step title="Review Output">
    Check a few files manually:
    
    ```bash
    less cleaned/01.md
    diff 01.md cleaned/01.md
    ```
  </Step>
  
  <Step title="Combine if Needed">
    Create final output:
    
    ```bash
    cat cleaned/*.md > ../final_book.md
    ```
  </Step>
</Steps>

## Common Issues and Solutions

<AccordionGroup>
  <Accordion title="Headers Not Being Removed">
    **Problem**: Some headers remain in the output.
    
    **Solution**: Check that the header text exactly matches what's in the code:
    
    ```javascript
    const headersAndFooters = [
      "Exact Header Text",  // Must match exactly
      "Another Header"
    ];
    ```
    
    Note: Matching is case-sensitive.
  </Accordion>
  
  <Accordion title="Page Numbers Remain">
    **Problem**: Page numbers aren't being removed.
    
    **Solution**: Check the page number format. The default patterns assume:
    - Standalone numbers on their own line
    - Standard Arabic numerals (1, 2, 3...)
    
    For different formats, modify the regex patterns in `clean_text.ts`.
  </Accordion>
  
  <Accordion title="Too Much Whitespace Removed">
    **Problem**: Paragraph breaks are lost.
    
    **Solution**: Check that your input has proper paragraph breaks (blank lines). CADS preserves existing paragraph structure.
  </Accordion>
  
  <Accordion title="Special Characters Cause Issues">
    **Problem**: Text with special regex characters causes errors.
    
    **Solution**: Escape special characters in header/footer definitions:
    
    ```javascript
    const headersAndFooters = [
      "Price: \\$9\\.99",  // Escape $ and .
      "Section \\[1\\]",   // Escape brackets
    ];
    ```
  </Accordion>
</AccordionGroup>
